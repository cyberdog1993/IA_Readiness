# Manual de usuario

## Objetivo

Guiar el uso diario de la plataforma de diagnóstico y levantamiento para Consultores IT.

## Perfiles

- Cliente: completa el diagnóstico, revisa el resultado y entra a su portal.
- Consultor interno: levanta procesos, registra evidencia y prepara propuestas.
- Administrador: gestiona usuarios, accesos, clientes y exportaciones.

## Flujo público

1. Ingresar a la landing pública.
2. Completar el diagnóstico rápido.
3. Revisar el puntaje de madurez, oportunidades y recomendación.
4. Dejar los datos de contacto para descargar el informe.
5. Solicitar consultoría o agendar una reunión.

## Flujo de consultoría

1. Iniciar sesión con una cuenta autorizada.
2. Entrar al portal cliente.
3. Abrir el proceso correspondiente.
4. Completar el levantamiento por secciones.
5. Descargar PDF, Markdown o JSON para revisión o para IA.

## Flujo administrativo

1. Entrar al panel `/admin`.
2. Crear o editar clientes y usuarios.
3. Revisar procesos y avances.
4. Generar exportaciones y propuestas preliminares.
5. Enviar payload a n8n o webhooks cuando sea necesario.

## Exportables

- PDF: informe legible para cliente o consultor.
- Markdown: ideal para ChatGPT, Claude y agentes.
- JSON: ideal para integraciones y automatización.

## Recomendaciones de uso

- Mantener un proceso por cliente cuando sea posible.
- Completar el levantamiento por partes para no perder información.
- Usar el portal cliente solo para sus propios procesos.
- Compartir el Markdown con un LLM para obtener resúmenes y propuestas preliminares.
